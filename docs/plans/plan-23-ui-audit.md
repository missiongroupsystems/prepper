### P1 ‚Äî Major

**[P1] Missing `aria-label` on hamburger button**
- **Location**: `frontend/src/components/layout/TopNav.tsx:131‚Äì136`
- **Category**: Accessibility
- **Impact**: Button has no accessible name ‚Äî screen readers say "button" with no context. WCAG 4.1.2 Name, Role, Value (Level AA).
- **Fix**: Add `aria-label={isMenuOpen ? "Close navigation" : "Open navigation"}` and `aria-expanded={isMenuOpen}`.
- **Command**: `/harden`

**[P1] Tab navigation missing ARIA roles ‚Äî 4 pages**
- **Location**: `recipes/page.tsx:31‚Äì46`, `rnd/page.tsx:795`, `ingredients/page.tsx:402`, `settings/page.tsx:50`
- **Category**: Accessibility
- **Impact**: Tab UIs using `<nav> + <button>` without `role="tablist"` / `role="tab"` / `aria-selected`. WCAG 4.1.2 (Level AA).
- **Fix**: Add `role="tablist"` on container, `role="tab"` + `aria-selected={isActive}` on each button, `role="tabpanel"` on content. Extract into reusable `<TabNav>` component to fix all 4 pages at once.
- **Command**: `/harden`

**[P1] Touch targets below 44px ‚Äî systemic**
- **Location**: `Button.tsx:34` (`size="sm"` = `h-8` = 32px), `menu-sketch/page.tsx:85` (delete ~22px), `TopNav.tsx:158` (icon-only nav ~36px)
- **Category**: Responsive Design
- **Impact**: Kitchen app = tablet/mobile use. Mis-taps on small icon buttons. WCAG 2.5.5 Target Size (Level AA).
- **Fix**: `size="sm"` ‚Üí `h-11 px-3` (44px min). Icon-only buttons ‚Üí `min-h-11 min-w-11`. Delete button in menu-sketch ‚Üí `p-2` with `h-4 w-4` icon.
- **Command**: `/adapt`

**[P1] Zinc color hardcoding throughout core surfaces**
- **Location**: `TopNav.tsx` (lines 102, 133‚Äì134, 154‚Äì155, 162, 179), `Modal.tsx` (lines 95, 108‚Äì109, 119, 126), `finance/page.tsx` (lines 53‚Äì100), `login/page.tsx` (lines 85, 98), `RecipeCard.tsx` (lines 34, 49, 73, 87, 101)
- **Category**: Theming
- **Impact**: `bg-white`, `bg-zinc-950`, `border-zinc-200`, `text-zinc-600` used instead of design tokens. Surfaces don't correctly respond to theme changes without duplicated `dark:` variants.
- **Fix**: `bg-white` ‚Üí `bg-background`, `bg-zinc-950` ‚Üí `bg-background`, `border-zinc-200` ‚Üí `border-border`, `text-zinc-600` ‚Üí `text-muted-foreground`. Eliminate `dark:` inline overrides on core surfaces.
- **Command**: `/normalize`

### P2 ‚Äî Minor

**[P2] `transition-all` in shared animation utility**
- **Location**: `globals.css:34` (`.flow-ui-hover-lift`)
- **Category**: Performance
- **Impact**: Transitions every CSS property unnecessarily, including layout properties.
- **Fix**: Replace `transition-all duration-300` with `transition-[transform,box-shadow] duration-300`.
- **Command**: `/optimize`

**[P2] Non-memoized callback functions in render scope**
- **Location**: `RecipeManagementTab.tsx:188‚Äì208` (`getOutletNamesForRecipe`, `getCategoryNamesForRecipe`, `getAllergenNamesForRecipe`)
- **Category**: Performance
- **Impact**: Functions recreated every render; all recipe cards re-render on search/filter changes with 30+ recipes per page.
- **Fix**: Wrap in `useCallback` with appropriate dependencies, or pre-compute lookup maps and access values directly in JSX.
- **Command**: `/optimize`

**[P2] `<a href>` instead of Next.js `<Link>` in menu sketch list**
- **Location**: `frontend/src/app/menu-sketch/page.tsx:65‚Äì68`
- **Category**: Performance
- **Impact**: Full page reload instead of client-side transition when clicking a sketch card. Loses React state, causes visible flicker.
- **Fix**: Replace `<a href={...}>` with `<Link href={...}>` from `next/link`.
- **Command**: `/optimize`

**[P2] Modal backdrop uses hardcoded `bg-black/50` instead of overlay token**
- **Location**: `Modal.tsx:95`
- **Category**: Theming
- **Impact**: Ignores `--dialog-overlay` token (defined at `globals.css:506`). Overlay contrast wrong in both themes.
- **Fix**: Replace `bg-black/50` with a utility built on `hsl(var(--dialog-overlay))`.
- **Command**: `/normalize`

**[P2] Toolbar select overflow risk at mid-screen widths**
- **Location**: `RecipeManagementTab.tsx:261‚Äì283` (sort + status + group selects in a flex row)
- **Category**: Responsive Design
- **Impact**: Four controls (`w-44 + w-32 + w-36 + ViewToggle`) may overflow between 640px‚Äì900px where the row becomes horizontal but screen isn't wide enough.
- **Fix**: Wrap controls in `flex-wrap` or move secondary filters to a collapsible row.
- **Command**: `/adapt`

## Systemic Issues

1. **Zinc hardcoding is pervasive** ‚Äî `text-zinc-*`, `bg-zinc-*`, `border-zinc-*` appear in 15+ components. Every core surface needs normalizing to design tokens. The token system exists and is complete ‚Äî it's just not being applied.

2. **Touch targets consistently undersized** ‚Äî `size="sm"` (32px) used broadly; icon action buttons at 22‚Äì28px. Kitchen/tablet context makes this high priority for the real audience.

3. **Tab navigation pattern used on 4+ pages without ARIA** ‚Äî identical anti-pattern. One reusable `<TabNav>` component fix propagates everywhere.
